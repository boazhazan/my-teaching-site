שיעור 4.1 — שפת SQL

בסיסי נתונים, פקודות CRUD, שאילתות ומחרוזת חיבור

1. מה זה בסיס נתונים?

בסיס נתונים (Database) הוא אוסף מאורגן של נתונים שנשמר בצורה מובנית, כך שקל לאחזר, לעדכן ולנהל אותם. בסיס נתונים רלציוני (Relational Database) מאחסן את המידע בטבלאות.

מבנה טבלה

  • שורות (Rows) = רשומות (Records) — כל שורה מייצגת פריט נתונים אחד
  • עמודות (Columns) = שדות (Fields) — כל עמודה מייצגת מאפיין של הנתונים
  • מפתח ראשי (Primary Key) — עמודה עם ערך ייחודי שמזהה כל שורה באופן חד-משמעי
💡 למה צריך מפתח ראשי?

כדי שנוכל לזהות כל שורה באופן ייחודי. למשל, אם יש שני תלמידים ששניהם "דני" — ה-Id הייחודי מבדיל ביניהם.

הנה דוגמה לטבלה Users:

כל הדוגמאות, השאילתות והסימולטור האינטראקטיבי עובדים מול הטבלה הזאת.

2. פקודות CRUD

SQL (Structured Query Language) היא שפה לניהול בסיסי נתונים. ארבע הפעולות הבסיסיות נקראות CRUD:

  • Create — יצירה (INSERT)
  • Read — קריאה (SELECT)
  • Update — עדכון (UPDATE)
  • Delete — מחיקה (DELETE)

R SELECT — קריאת נתונים

הפקודה הנפוצה ביותר. מחזירה נתונים מהטבלה לפי תנאים.

שליפת כל הנתונים:

SELECT * FROM Users

הכוכבית (*) אומרת "כל העמודות".

שליפת עמודות ספציפיות עם תנאי:

SELECT Name, Age FROM Users WHERE Age > 16

מחזיר רק את השם והגיל של מי שגילו מעל 16.

מיון תוצאות:

SELECT * FROM Users ORDER BY Age DESC

ORDER BY ממיין — ASC מהקטן לגדול (ברירת מחדל), DESC מהגדול לקטן.

פונקציות צבירה (Aggregate):

SELECT COUNT(*) FROM Users

סופר את מספר השורות. מחזיר 5.

SELECT City, COUNT(*) AS Total FROM Users GROUP BY City

GROUP BY מקבץ לפי עמודה, ו-COUNT(*) סופר כמה שורות בכל קבוצה.

SELECT AVG(Age) FROM Users

מחשב את הגיל הממוצע.

C INSERT — הוספת שורה

מוסיפה רשומה חדשה לטבלה:

INSERT INTO Users (Name, Age, City) VALUES ('שרה', 17, 'אילת')

שימו לב: לא צריך לציין את Id — הוא אוטומטי (Auto-Increment).

⚠️ שימו לב לתחביר

מחרוזות ב-SQL עטופות בגרשיים בודדים ('), לא כפולים ("). מספרים נכתבים בלי גרשיים.

U UPDATE — עדכון נתונים

משנה ערכים קיימים בטבלה:

UPDATE Users SET Age = 17 WHERE Name = 'דני'

מעדכן את הגיל של דני ל-17.

⚠️ אזהרה חשובה!

בלי WHERE, הפקודה תעדכן את כל השורות! תמיד תוודאו שיש תנאי.

D DELETE — מחיקת שורה

מסירה רשומה מהטבלה:

DELETE FROM Users WHERE Id = 5

מוחקת את השורה שבה Id שווה 5 (יובל).

⚠️ זהירות כפולה!

בלי WHERE, הפקודה תמחק את כל השורות בטבלה! אין Undo.

3. Connection String — מחרוזת חיבור

כדי שהאפליקציה שלנו (C# / ASP.NET) תוכל לתקשר עם בסיס הנתונים, היא צריכה מחרוזת חיבור (Connection String) — כתובת שמכילה את כל הפרטים לחיבור.

מה מכילה מחרוזת חיבור?

  • Server — שם השרת (למשל localhost בזמן פיתוח)
  • Database — שם בסיס הנתונים
  • Authentication — שם משתמש וסיסמה (או Windows Authentication)
"Server=localhost;Database=MySchoolDB;Trusted_Connection=True;"
💡 Trusted_Connection=True

אומר לשרת להשתמש באימות Windows — לא צריך שם משתמש וסיסמה נפרדים. מאוד נוח בסביבת פיתוח.
