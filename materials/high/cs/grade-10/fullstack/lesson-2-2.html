1. תנאים (Conditions)

בשיעור הקודם למדנו להגדיר משתנים ולשנות תוכן בדף. אבל תוכנית אמיתית צריכה לקבל החלטות — לעשות דבר אחד אם התנאי מתקיים, ודבר אחר אם לא. לזה בדיוק משמשת פקודת if.

if / else if / else

המבנה הבסיסי ביותר לבדיקת תנאי:

let grade = 85; if (grade >= 90) { console.log("מעולה! ציון גבוה מאוד"); } else if (grade >= 70) { console.log("טוב מאוד!"); } else if (grade >= 56) { console.log("עברת, אבל אפשר להשתפר"); } else { console.log("נכשלת. נסה שוב!"); }

איך זה עובד?

  • הדפדפן בודק את התנאי הראשון (grade >= 90). אם הוא אמת (true) — מריץ את הבלוק שלו ומדלג על כל השאר.
  • אם הראשון שקר (false) — עובר לבדוק את ה-else if הבא.
  • אם אף תנאי לא התקיים — מריץ את בלוק ה-else (אם קיים).
💡 אופרטורי השוואה

=== שווה (השוואה מדויקת), !== לא שווה, > גדול מ-, < קטן מ-, >= גדול-שווה, <= קטן-שווה. תמיד עדיף === על == כי הוא בודק גם סוג נתונים.

switch-case

כשאנחנו רוצים להשוות משתנה לכמה ערכים קבועים, switch הוא נקי יותר מ-if:

let day = "שני"; switch(day) { case "ראשון": console.log("תחילת השבוע"); break; case "שישי": console.log("סוף שבוע!"); break; case "שבת": console.log("יום מנוחה"); break; default: console.log("יום רגיל בשבוע"); }

שימו לב:

  • כל case בודק אם הערך שווה למשתנה.
  • break עוצר את הבדיקה — בלעדיו, הקוד ימשיך ל-case הבא! (זו שגיאה נפוצה)
  • default — כמו else, מופעל כשאף case לא תאם.
⚠️ שכחתם break?

אם תשכחו לכתוב break אחרי case, הקוד "ייפול" (fall-through) ל-case הבא ויריץ גם אותו. זו שגיאה קלאסית ב-JavaScript — תמיד לסיים כל case ב-break.

2. לולאות (Loops)

לולאה מאפשרת לנו להריץ קוד שוב ושוב בלי לכתוב אותו כמה פעמים. למשל, אם רוצים להדפיס את המספרים 1 עד 100 — לא נכתוב 100 שורות console.log, אלא לולאה אחת.

לולאת for — כשיודעים כמה פעמים

לולאת for מתאימה כשאנחנו יודעים מראש כמה חזרות אנחנו רוצים:

// מבנה: for (אתחול; תנאי; קידום) for (let i = 1; i <= 10; i++) { console.log("מספר: " + i); }

שלושה חלקים בסוגריים, מופרדים בנקודה-פסיק:

  1. אתחול (let i = 1) — רץ פעם אחת בהתחלה. מגדיר את מונה הלולאה.
  2. תנאי (i <= 10) — נבדק לפני כל חזרה. כל עוד true — הלולאה ממשיכה.
  3. קידום (i++) — רץ אחרי כל חזרה. בדרך כלל מגדיל את המונה.

דוגמה — לוח הכפל של 7:

for (let i = 1; i <= 10; i++) { let result = 7 * i; console.log(7 + " x " + i + " = " + result); } // 7 x 1 = 7 // 7 x 2 = 14 // ... // 7 x 10 = 70

לולאת while — כשלא יודעים כמה פעמים

לולאת while ממשיכה כל עוד התנאי מתקיים, בלי מונה מוגדר מראש:

let num = 1; while (num <= 100) { console.log(num); num = num * 2; // כפול 2 בכל פעם } // 1, 2, 4, 8, 16, 32, 64
⚠️ זהירות מלולאה אינסופית!

אם שוכחים לעדכן את המשתנה בתוך הלולאה (או שהתנאי תמיד true), הלולאה תרוץ לנצח והדפדפן ייתקע. תמיד וודאו שהתנאי יהפוך ל-false בשלב כלשהו.

break ו-continue

שתי פקודות שליטה שימושיות בתוך לולאות:

// break — עוצר את הלולאה לחלוטין for (let i = 1; i <= 100; i++) { if (i === 5) { break; // יוצאים מהלולאה כשi=5 } console.log(i); } // ידפיס: 1, 2, 3, 4 // continue — מדלג על החזרה הנוכחית for (let i = 1; i <= 10; i++) { if (i % 2 === 0) { continue; // מדלג על מספרים זוגיים } console.log(i); } // ידפיס: 1, 3, 5, 7, 9 (רק אי-זוגיים)
✅ מתי להשתמש במה?

for — כשיודעים מראש כמה חזרות (מספרים 1 עד 10, רשימה בגודל ידוע). while — כשלא יודעים (חכה עד שהמשתמש מזין ערך נכון, חפש עד שתמצא).

3. פונקציות (Functions)

פונקציה היא בלוק קוד עם שם שאפשר להפעיל (לקרוא לו) כמה פעמים שרוצים. במקום לכתוב את אותו קוד שוב ושוב, כותבים אותו פעם אחת בפונקציה וקוראים לה בכל פעם שצריך.

הגדרת פונקציה

// הגדרה: function שם(פרמטרים) { קוד } function greet(name) { console.log("שלום, " + name + "!"); } // קריאה (הפעלה): greet("יוסי"); // "שלום, יוסי!" greet("דנה"); // "שלום, דנה!" greet("עומר"); // "שלום, עומר!"
  • פרמטר (name) — משתנה שהפונקציה מקבלת מבחוץ. אפשר כמה פרמטרים, מופרדים בפסיקים.
  • קריאה — שם הפונקציה + סוגריים עם הערכים (ארגומנטים).

ערך החזרה — return

פונקציה יכולה גם להחזיר ערך למי שקרא לה:

function calcArea(width, height) { let area = width * height; return area; } let result = calcArea(5, 3); console.log("השטח: " + result); // "השטח: 15" // אפשר גם ישירות: console.log("שטח: " + calcArea(10, 7)); // "שטח: 70"

return עושה שני דברים: מחזיר ערך ויוצא מהפונקציה. כל קוד שאחרי return לא ירוץ.

למה פונקציות?

  • מניעת כפילות — כותבים קוד פעם אחת, קוראים לו מכל מקום.
  • סדר וארגון — כל פונקציה עושה דבר אחד מוגדר, קל לקרוא ולתחזק.
  • בדיקות קלות — אפשר לבדוק כל פונקציה בנפרד.

Arrow Functions (אזכור קצר)

ב-JavaScript מודרני יש גם תחביר מקוצר לפונקציות, נקרא Arrow Function:

// פונקציה רגילה: function double(x) { return x * 2; } // Arrow function — אותו דבר בדיוק, רק קצר יותר: const double = (x) => x * 2; // דוגמה נוספת: const add = (a, b) => a + b; console.log(add(3, 4)); // 7
💡 טיפ

Arrow functions נפוצות מאוד בקוד מודרני. אין חובה להשתמש בהן עכשיו — אבל כדאי להכיר את התחביר כי תראו אותו הרבה בפרויקטים אמיתיים.

4. אירועים (Events)

אירוע (Event) הוא כל דבר שקורה בדף — המשתמש לוחץ על כפתור, מזיז את העכבר, משנה ערך בשדה, שולח טופס. JavaScript מאפשר לנו "להאזין" לאירועים ולהגיב אליהם — וזה מה שהופך דף סטטי לאפליקציה אינטראקטיבית.

onclick — לחיצה

האירוע הנפוץ ביותר. מופעל כשלוחצים על אלמנט:

<button onclick="sayHello()">לחץ עליי</button> <script> function sayHello() { alert("שלום! לחצת על הכפתור"); } </script>

כשהמשתמש לוחץ על הכפתור, הדפדפן קורא לפונקציה sayHello(). זה חיבור ישיר בין אירוע (לחיצה) לפעולה (הפונקציה).

onchange — שינוי ערך

מופעל כשהמשתמש משנה ערך ב-select, input, או textarea:

<select onchange="colorChanged(this.value)"> <option value="red">אדום</option> <option value="blue">כחול</option> <option value="green">ירוק</option> </select> <script> function colorChanged(color) { document.body.style.backgroundColor = color; console.log("צבע הרקע שונה ל: " + color); } </script>

this.value מעביר את הערך הנבחר ישירות לפונקציה. כך הפונקציה יודעת איזה צבע נבחר.

onsubmit — שליחת טופס

מופעל כשהמשתמש שולח טופס. חשוב במיוחד כי מאפשר לנו לבדוק את הנתונים לפני השליחה:

<form onsubmit="return validateForm()"> <input type="text" id="username" placeholder="שם משתמש"> <button type="submit">שלח</button> </form> <script> function validateForm() { let name = document.getElementById("username").value; if (name === "") { alert("אנא הזן שם משתמש!"); return false; // מונע שליחה } return true; // מאפשר שליחה } </script>

שימו לב ל-return ב-onsubmit — אם הפונקציה מחזירה false, הטופס לא יישלח. כך אפשר לבדוק שכל השדות תקינים.

onmouseover / onmouseout — ריחוף עכבר

onmouseover מופעל כשהעכבר נכנס לאלמנט, ו-onmouseout כשהוא עוזב:

<div id="box" onmouseover="this.style.background='yellow'" onmouseout="this.style.background='lightgray'" style="padding:40px;text-align:center;background:lightgray"> רחף עליי! </div>

addEventListener — הדרך המתקדמת

בדוגמאות למעלה השתמשנו ב-inline events (כתבנו את האירוע ישירות ב-HTML). בפרויקטים גדולים, עדיף להפריד את ה-JavaScript מה-HTML באמצעות addEventListener:

<button id="myBtn">לחץ</button> <script> // במקום onclick="..." ב-HTML: let btn = document.getElementById("myBtn"); btn.addEventListener("click", function() { alert("לחצת!"); }); </script>

היתרון: אפשר להוסיף כמה מאזינים לאותו אלמנט, וה-HTML נשאר נקי. זו השיטה המומלצת בפרויקטים אמיתיים.

✅ סיכום אירועים

onclick — לחיצה, onchange — שינוי ערך, onsubmit — שליחת טופס, onmouseover/onmouseout — ריחוף. בכל המקרים: מחברים אירוע לפונקציה שתרוץ כשהאירוע קורה.

📝 תרגיל: בניית מחשבון מתקדם

השתמשו בעורך הקוד למעלה ובנו מחשבון פונקציונלי עם הדרישות הבאות:

  1. תצוגת מסך — שדה שמציג את הביטוי הנוכחי ואת התוצאה (כמו מחשבון אמיתי)
  2. כפתורי ספרות — כפתורים 0-9 שמוסיפים את הספרה לתצוגה (השתמשו ב-onclick)
  3. כפתורי פעולות — כפתורים ל- +, -, *, /
  4. כפתור "=" — מחשב את התוצאה (רמז: אפשר להשתמש ב-eval() או לבנות לוגיקה עם switch)
  5. כפתור "C" — מנקה את התצוגה
  6. פריסה בטבלת HTML — סדרו את הכפתורים בטבלה (4 עמודות כמו מחשבון אמיתי)

בונוס: הוסיפו כפתור נקודה עשרונית, כפתור +/- להפיכת סימן, ו-CSS לעיצוב (צבעי כפתורים שונים לספרות ולפעולות).
