עבודה בשיטה המקושרת (Connected)

ExecuteNonQuery, ExecuteScalar ואבטחה מפני SQL Injection

1. שיטת עבודה Connected

בשיעור הקודם למדנו את שיטת העבודה הלא מקושרת (Disconnected) — שבה מביאים את כל הנתונים לזיכרון המחשב באמצעות DataSet ו-DataAdapter, ורק אז עובדים איתם. השיטה הזו מתאימה לקריאת נתונים (SELECT).

עכשיו נלמד את שיטת העבודה המקושרת (Connected). בשיטה הזו, אנחנו פותחים חיבור ישיר לבסיס הנתונים, מבצעים את הפעולה, וסוגרים את החיבור מיד. השיטה הזו מתאימה לפעולות שמשנות נתונים: INSERT, UPDATE ו-DELETE.

מה ההבדל בין שתי השיטות?

Connected שינוי נתונים (INSERT / UPDATE / DELETE) Execute() int — מספר שורות שהושפעו

למה Connected ולא Disconnected לשינויים?

כש-INSERT/UPDATE/DELETE — אין צורך להביא נתונים לזיכרון. אנחנו רק שולחים פקודה לDB ומקבלים חזרה מספר (כמה שורות הושפעו). לכן חיבור ישיר יותר יעיל ופשוט.

2. ExecuteNonQuery

המתודה ExecuteNonQuery משמשת לביצוע פקודות SQL שלא מחזירות נתונים — כלומר INSERT, UPDATE ו-DELETE. היא מחזירה int — מספר השורות שהושפעו מהפקודה.

מתודת Execute במחלקת Helper

public static int Execute(string query) { using (SqlConnection conn = new SqlConnection(connStr)) { conn.Open(); SqlCommand cmd = new SqlCommand(query, conn); return cmd.ExecuteNonQuery(); } }

בואו נפרק את הקוד שורה אחרי שורה:

  • using (SqlConnection conn = ...) — פותח חיבור לDB. בלוק using מבטיח שהחיבור ייסגר אוטומטית בסוף, גם אם יש שגיאה.
  • conn.Open() — פותח את החיבור בפועל. שימו לב: בשיטה ה-Disconnected, Fill() פתח וסגר את החיבור אוטומטית. כאן אנחנו חייבים לפתוח ידנית עם Open().
  • SqlCommand cmd = new SqlCommand(query, conn) — יוצר אובייקט פקודה שמכיל את שאילתת ה-SQL ואת החיבור שדרכו תבוצע.
  • cmd.ExecuteNonQuery() — מבצע את הפקודה ומחזיר כמה שורות הושפעו.

דוגמת שימוש

// הוספת משתמש חדש int rows = Helper.Execute( "INSERT INTO Users (Name, Age) VALUES ('דני', 16)"); if (rows > 0) Message = "המשתמש נוסף בהצלחה!"; else Message = "שגיאה בהוספה";
כלל אצבע

אם ExecuteNonQuery מחזיר 0 — הפקודה הצליחה אבל אף שורה לא הושפעה (למשל, WHERE לא מצא התאמה). אם מחזיר מספר חיובי — כמות השורות שנוספו/עודכנו/נמחקו.

דוגמאות נוספות

// עדכון גיל משתמש int rows = Helper.Execute( "UPDATE Users SET Age = 17 WHERE Name = 'דני'"); // מחיקת משתמש int rows = Helper.Execute( "DELETE FROM Users WHERE Id = 5");

3. ExecuteScalar

לפעמים אנחנו צריכים לקבל מהDB ערך בודד — תא אחד בלבד. למשל: כמה משתמשים רשומים? האם שם המשתמש כבר תפוס? האם הסיסמה נכונה? בשביל זה יש את ExecuteScalar.

ExecuteScalar מחזירה object — את הערך של התא הראשון בשורה הראשונה בתוצאה. אם אין תוצאה — מחזירה null.

מתודת Scalar במחלקת Helper

public static object Scalar(string query) { using (SqlConnection conn = new SqlConnection(connStr)) { conn.Open(); SqlCommand cmd = new SqlCommand(query, conn); return cmd.ExecuteScalar(); } }

שימו לב שהמבנה כמעט זהה ל-Execute — ההבדל היחיד הוא שכאן אנחנו קוראים ל-ExecuteScalar() במקום ExecuteNonQuery(), ומחזירים object במקום int.

דוגמת שימוש: ספירת משתמשים

// כמה משתמשים יש בטבלה? int count = (int)Helper.Scalar( "SELECT COUNT(*) FROM Users"); Message = "יש " + count + " משתמשים רשומים";
חובה לבצע המרה (Casting)!

מכיוון ש-Scalar מחזירה object, חייבים להמיר לטיפוס הנכון. COUNT(*) מחזיר int, לכן נכתוב (int)Helper.Scalar(...). אם שוכחים את ההמרה — תהיה שגיאת compilation.

דוגמת שימוש: בדיקת Login

// בדיקת כניסה — האם שם המשתמש והסיסמה קיימים? object result = Helper.Scalar( "SELECT COUNT(*) FROM Users " + "WHERE Username='admin' AND Password='1234'"); if ((int)result > 0) Message = "התחברת בהצלחה!"; else Message = "שם משתמש או סיסמה שגויים";

הרעיון: שולחים שאילתה שסופרת כמה שורות מתאימות. אם התוצאה גדולה מ-0 — המשתמש קיים והסיסמה נכונה.

סיכום שלוש המתודות

מתודהמה עושהמחזירהמתי?
GetData(query) מביאה טבלת נתונים DataTable SELECT — הצגת נתונים
Execute(query) מבצעת פקודה ללא החזרת נתונים int INSERT / UPDATE / DELETE
Scalar(query) מחזירה ערך בודד object COUNT, Login, בדיקת קיום

4. אבטחה — SQL Injection

זהו אחד מנושאי האבטחה הקריטיים ביותר בפיתוח ווב. הבנה של ההתקפה הזו והגנה מפניה היא חובה של כל מפתח.

מה זה SQL Injection?

SQL Injection (הזרקת SQL) היא התקפה שבה תוקף מזין קוד SQL זדוני לתוך שדות קלט (כמו שדה שם משתמש בטופס התחברות), במטרה לגרום לשרת להריץ שאילתות שלא תוכננו מראש.

דוגמה להתקפה

נניח שיש לנו דף Login פשוט שמקבל שם משתמש וסיסמה ובודק אותם מול בסיס הנתונים:

// קוד פגיע! אל תכתבו ככה! string query = "SELECT * FROM Users WHERE Username='" + username + "' AND Password='" + password + "'";

אם המשתמש מזין שם משתמש רגיל כמו admin וסיסמה 1234, השאילתה תיראה כך:

SELECT * FROM Users WHERE Username='admin' AND Password='1234'

אבל מה קורה אם תוקף מזין בשדה שם המשתמש את הערך הבא?

' OR 1=1 --

השאילתה שתיווצר תיראה כך:

SELECT * FROM Users WHERE Username='' OR 1=1 --' AND Password='...'

מה קרה כאן?

  • ' OR 1=1 — סוגר את המחרוזת של ה-Username, ומוסיף תנאי שתמיד אמיתי (1=1)
  • -- — הערה (comment) ב-SQL, שמבטלת את כל המשך השאילתה (כולל בדיקת הסיסמה!)
  • התוצאה: השאילתה מחזירה את כל המשתמשים — והתוקף מצליח להתחבר בלי סיסמה!
SQL Injection מאפשר לתוקף:

לקרוא מידע רגיש (סיסמאות, פרטים אישיים), לשנות נתונים, למחוק טבלאות שלמות, ובמקרים קיצוניים — להשתלט על השרת. זו אחת מ-10 ההתקפות הנפוצות ביותר באינטרנט (OWASP Top 10).

השוואה: קוד פגיע מול קוד מאובטח

❌ קוד פגיע (שרשור מחרוזות)

string query = "SELECT * FROM Users " + "WHERE Username='" + username + "'";

הקלט מוזרק ישירות לשאילתה!

✅ קוד מאובטח (Parameters)

string query = "SELECT * FROM Users " + "WHERE Username=@name"; cmd.Parameters.AddWithValue( "@name", username);

הקלט עובר סינון אוטומטי!

איך Parameters מגינים?

כשמשתמשים ב-Parameters (פרמטרים), ה-SQL Server מפריד בין מבנה השאילתה לבין הערכים. גם אם התוקף מזין ' OR 1=1 --, SQL Server יתייחס לזה כמחרוזת טקסט רגילה ולא כקוד SQL. הערך פשוט לא יימצא בטבלה — והשאילתה תחזיר 0 תוצאות.

כלל הזהב של אבטחה:

לעולם אל תשרשרו קלט משתמש ישירות לשאילתת SQL! תמיד השתמשו בפרמטרים (@param + AddWithValue). זה חל על כל מקום שיש בו קלט — טפסים, URL, קבצי Cookie ועוד.

מתודות Helper מעודכנות עם Parameters

כעת נעדכן את המתודות שלנו כך שיתמכו בפרמטרים:

public static int Execute(string query, params SqlParameter[] parameters) { using (SqlConnection conn = new SqlConnection(connStr)) { conn.Open(); SqlCommand cmd = new SqlCommand(query, conn); cmd.Parameters.AddRange(parameters); return cmd.ExecuteNonQuery(); } }
public static object Scalar(string query, params SqlParameter[] parameters) { using (SqlConnection conn = new SqlConnection(connStr)) { conn.Open(); SqlCommand cmd = new SqlCommand(query, conn); cmd.Parameters.AddRange(parameters); return cmd.ExecuteScalar(); } }

דוגמת שימוש מאובטחת

// Login מאובטח עם Parameters string query = "SELECT COUNT(*) FROM Users " + "WHERE Username=@user AND Password=@pass"; object result = Helper.Scalar(query, new SqlParameter("@user", username), new SqlParameter("@pass", password)); if ((int)result > 0) Message = "התחברת בהצלחה!"; else Message = "שם משתמש או סיסמה שגויים";
