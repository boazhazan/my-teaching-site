שיעור 4.2 — עבודה בשיטה הלא מקושרת (Disconnected)

DataTable, מחלקת Helper ושליפת נתונים ב-Razor Pages

1. שיטת עבודה Disconnected

בשיעור הקודם למדנו איך להתחבר לבסיס נתונים באמצעות SqlConnection. עכשיו נלמד את השיטה הלא מקושרת — הגישה המועדפת ברוב האפליקציות האמיתיות.

העיקרון: חיבור קצר ומהיר

במקום להחזיק חיבור פתוח לבסיס הנתונים לאורך כל חיי הדף, אנחנו פועלים בשלושה שלבים:

  1. מתחברים לבסיס הנתונים
  2. שולפים את הנתונים לזיכרון (לתוך DataTable)
  3. מנתקים את החיבור — וממשיכים לעבוד עם הנתונים בזיכרון

תרשים הזרימה

האפליקציה שלנו
↓
Open Connection
↓
SqlDataAdapter.Fill(DataTable)
↓
Close Connection
↓
עבודה עם DataTable בזיכרון

שימו לב: החיבור לבסיס הנתונים פתוח רק לרגע קצר — רק בזמן שליפת הנתונים. אחרי שהנתונים נמצאים בזיכרון (בתוך DataTable), אפשר לעבוד איתם חופשי בלי שום תלות בחיבור.

למה זה טוב?

  • פחות עומס על השרת — חיבור פתוח צורך משאבים. ככל שנסגור אותו מהר יותר, כך השרת יכול לשרת יותר משתמשים
  • ביצועים טובים יותר — הנתונים כבר בזיכרון, אין צורך לפנות לדיסק בכל קריאה
  • אמינות — אם החיבור נופל באמצע, הנתונים שכבר נשלפו עדיין זמינים
📚
אנלוגיה: חשבו על ספרייה ציבורית. בשיטה המקושרת (Connected) — אתם יושבים בספרייה, קוראים את הספר, וכל עוד אתם קוראים — אתם תופסים כיסא ומקום. בשיטה הלא מקושרת (Disconnected) — אתם לוקחים את הספר הביתה, מחזירים את הכיסא, וקוראים בנחת בבית. הספרנית פנויה לשרת אנשים אחרים!

המחלקות המרכזיות

  • DataTable — טבלה בזיכרון. מכילה שורות (DataRow) ועמודות (DataColumn), בדיוק כמו טבלה בבסיס נתונים
  • DataSet — אוסף של כמה DataTable (שימושי כשצריך כמה טבלאות ביחד)
  • SqlDataAdapter — ה"גשר" בין בסיס הנתונים ל-DataTable. הוא מבצע את השאילתה וממלא את הטבלה
נקודת מפתח

SqlDataAdapter פותח וסוגר את החיבור אוטומטית בזמן הקריאה ל-Fill(). אנחנו לא חייבים לעשות Open() ו-Close() ידנית!

2. מחלקת Helper

בפרויקט אמיתי, לא נרצה לכתוב את אותו קוד שליפה בכל דף מחדש. במקום זה, ניצור מחלקה סטטית אחת שמרכזת את כל הגישה לבסיס הנתונים. זו מחלקת Helper.

למה מחלקה סטטית?

  • מניעת שכפול קוד — כותבים פעם אחת, משתמשים בכל הדפים
  • תחזוקה קלה — אם צריך לשנות את מחרוזת ההתחברות, משנים במקום אחד
  • קריאות — הקוד בדפים הופך פשוט ונקי

הקוד המלא

public static class Helper { static string connStr = "Data Source=...;Initial Catalog=...;Integrated Security=True"; public static DataTable GetData(string query) { DataTable dt = new DataTable(); using (SqlConnection conn = new SqlConnection(connStr)) { SqlDataAdapter da = new SqlDataAdapter(query, conn); da.Fill(dt); } return dt; } }

הסבר שורה אחר שורה

public static class Helper
מחלקה סטטית — לא צריך ליצור מופע (new). קוראים ישירות: Helper.GetData(...)
static string connStr = "...";
מחרוזת התחברות — כתובת השרת, שם בסיס הנתונים, ואופן ההזדהות. מוגדרת פעם אחת ומשמשת את כל המתודות
public static DataTable GetData(string query)
המתודה הראשית — מקבלת שאילתת SQL כמחרוזת, מחזירה DataTable עם התוצאות
DataTable dt = new DataTable();
יוצרים טבלה ריקה בזיכרון — היא תתמלא בנתונים
using (SqlConnection conn = ...)
בלוק using — מבטיח שהחיבור ייסגר אוטומטית בסוף הבלוק, גם אם מתרחשת שגיאה!
SqlDataAdapter da = new ...(query, conn);
יוצרים מתאם נתונים — הוא יודע איזו שאילתה להריץ ודרך איזה חיבור
da.Fill(dt);
הפקודה המרכזית! מריצה את השאילתה, פותחת חיבור, שולפת נתונים, ממלאת את הטבלה, וסוגרת חיבור
return dt;
מחזירים את הטבלה המלאה — עכשיו אפשר לעבוד עם הנתונים בלי חיבור פעיל
חשוב לזכור!

using מבטיח שהחיבור ייסגר אוטומטית — גם אם נזרקת שגיאה באמצע הקוד. בלי using, חיבור שלא נסגר עלול "לדלוף" ולגרום לבעיות ביצועים חמורות. תמיד עטפו חיבורים ב-using!

3. שימוש בדף — Razor Pages

עכשיו שיש לנו מחלקת Helper, בואו נראה איך משתמשים בה בתוך דף Razor כדי להציג נתונים מבסיס הנתונים בטבלת HTML.

הקוד בצד השרת (.cshtml.cs)

// IndexModel.cshtml.cs public DataTable UsersTable { get; set; } public void OnGet() { UsersTable = Helper.GetData("SELECT * FROM Users"); }

מה קורה כאן? כשהמשתמש נכנס לדף (בקשת GET), המתודה OnGet() נקראת אוטומטית. היא קוראת ל-Helper.GetData עם שאילתת SQL, ומקבלת DataTable עם כל המשתמשים.

הקוד בצד הלקוח (.cshtml)

<!-- Index.cshtml --> <table> <tr> <th>שם</th> <th>גיל</th> <th>עיר</th> </tr> @foreach (DataRow row in Model.UsersTable.Rows) { <tr> <td>@row["Name"]</td> <td>@row["Age"]</td> <td>@row["City"]</td> </tr> } </table>

לולאת @foreach עוברת על כל שורה ב-DataTable ויוצרת שורת HTML עבור כל משתמש. הסימן @ מסמן ל-Razor שזהו קוד C# שצריך להריץ.

התוצאה בדפדפן

נניח שטבלת Users בבסיס הנתונים נראית כך:

NameAgeCity

שליפה עם סינון (WHERE)

אפשר לשלוף רק חלק מהנתונים באמצעות WHERE:

// שליפת משתמשים מחיפה בלבד UsersTable = Helper.GetData("SELECT * FROM Users WHERE City = 'חיפה'"); // שליפת משתמשים מעל גיל 25 UsersTable = Helper.GetData("SELECT * FROM Users WHERE Age > 25"); // שליפה ממוינת לפי גיל UsersTable = Helper.GetData("SELECT * FROM Users ORDER BY Age");

הזרימה המלאה

משתמש נכנס לדף
←
OnGet() נקראת
←
Helper.GetData()
←
DB → DataTable
←
HTML מוצג
סיכום הזרימה

1. המשתמש גולש לדף → 2. OnGet() נקראת → 3. Helper.GetData() מתחברת ל-DB, שולפת ומנתקת → 4. הנתונים מוחזרים ב-DataTable → 5. Razor מציג אותם בטבלת HTML.
