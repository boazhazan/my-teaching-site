יחידה 2: JavaScript

אימות טפסים (Form Validation)

בדיקת קלט משתמש בצד הלקוח — שם משתמש, אימייל, סיסמה וטלפון

1. למה לבדוק בצד לקוח?

כשמשתמש ממלא טופס באתר (הרשמה, יצירת קשר, הזמנה), התוכן שהוא מזין לא תמיד תקין. אולי הוא שכח למלא שדה חובה, הזין אימייל בפורמט שגוי, או בחר סיסמה חלשה מדי. אימות טפסים (Form Validation) הוא התהליך שבו אנחנו בודקים את הקלט לפני שהוא נשלח.

אימות בצד הלקוח (Client-Side Validation) — כלומר בדיקה ש-JavaScript מבצעת בדפדפן — מספק שני יתרונות משמעותיים:

  • חוסך פנייה מיותרת לשרת — במקום לשלוח את הנתונים לשרת, לחכות לתשובה, ורק אז לגלות שהאימייל לא תקין — אפשר לזהות את הבעיה מיד, לפני השליחה.
  • חוויית משתמש טובה יותר — המשתמש מקבל משוב מיידי ויודע בדיוק מה צריך לתקן. לא צריך לחכות לטעינת דף חדש עם הודעת שגיאה.
⚠️ אזהרה חשובה!

ולידציה בצד הלקוח היא בדיקה ראשונית בלבד. השרת חייב לבדוק שוב את כל הנתונים! למה? כי כל מי שמבין JavaScript יכול לעקוף את הבדיקות בדפדפן (למשל דרך ה-DevTools). לעולם אל תסמוך רק על ולידציה בצד הלקוח.

אז למה בכלל לבדוק בצד הלקוח אם השרת גם בודק? כי זה משפר את חוויית השימוש. רוב המשתמשים הם לגיטימיים, והם יעריכו משוב מהיר. הבדיקה בשרת היא רשת הביטחון.

2. חוקי אימות

לפני שכותבים קוד, צריך להגדיר חוקים — מה נחשב קלט תקין לכל שדה. הנה חוקים אופייניים לטופס הרשמה:

שדהחוקי אימותדוגמה תקינה
שם משתמש חייב להתחיל באות, לפחות 3 תווים david123
סיסמה 6-12 תווים, חייבת ספרה + תו מיוחד Pass@5x
אימייל פורמט תקין (user@domain.com) david@school.co.il
טלפון 10 ספרות, מתחיל ב-05 0521234567

בואו נראה איך מתרגמים כל חוק לבדיקה ב-JavaScript:

בדיקת שם משתמש

function isValidUsername(value) { // חייב להתחיל באות (גדולה או קטנה) // אחריה לפחות 2 תווים נוספים (סה"כ 3+) if (value.length < 3) return false; let firstChar = value[0]; if (!((firstChar >= 'a' && firstChar <= 'z') || (firstChar >= 'A' && firstChar <= 'Z'))) { return false; } return true; }

בדיקת סיסמה

function isValidPassword(value) { // אורך: 6-12 תווים if (value.length < 6 || value.length > 12) return false; // חייבת לכלול לפחות ספרה אחת let hasDigit = false; for (let ch of value) { if (ch >= '0' && ch <= '9') hasDigit = true; } // חייבת לכלול תו מיוחד let specials = "!@#$%^&*"; let hasSpecial = false; for (let ch of value) { if (specials.includes(ch)) hasSpecial = true; } return hasDigit && hasSpecial; }

הגישה הזו עובדת, אבל הקוד ארוך ומסורבל. בסעיף הבא נלמד כלי הרבה יותר חזק ותמציתי: ביטויים רגולריים (RegEx).

3. ביטויים רגולריים (RegEx)

ביטוי רגולרי (Regular Expression, או בקיצור RegEx) הוא תבנית (pattern) לבדיקת טקסט. במקום לכתוב לולאות ותנאים ארוכים, אפשר להגדיר תבנית בשורה אחת שאומרת "הטקסט חייב להיראות ככה".

ב-JavaScript כותבים ביטוי רגולרי בין שני סלאשים /:

let pattern = /^[a-zA-Z]/; // מתחיל באות

תבניות חשובות

תבניתמשמעותדוגמה
/^[a-zA-Z]/ מתחיל באות (אנגלית) "david" ✔   "123abc" ✘
/[!@#$%]/ מכיל תו מיוחד "hi@5" ✔   "hello" ✘
/^\d{10}$/ בדיוק 10 ספרות "0521234567" ✔   "052123" ✘
/^[\w.-]+@[\w.-]+\.\w{2,}$/ פורמט אימייל "a@b.co" ✔   "abc" ✘

פירוק הסימנים

  • ^ — תחילת המחרוזת ("חייב להתחיל ב...")
  • $ — סוף המחרוזת ("חייב להסתיים ב...")
  • [a-zA-Z] — תו אחד מתוך הטווח (אות קטנה או גדולה)
  • \d — ספרה (0-9), שווה ערך ל-[0-9]
  • \w — תו "מילה" — אות, ספרה, או קו תחתון
  • {N} — בדיוק N פעמים. {2,} = לפחות 2 פעמים
  • + — פעם אחת או יותר
  • . — כל תו (חוץ מירידת שורה). לנקודה ממש: \.

שימוש ב-test()

המתודה regex.test(string) מחזירה true אם המחרוזת תואמת את התבנית, ו-false אחרת:

let emailPattern = /^[\w.-]+@[\w.-]+\.\w{2,}$/; emailPattern.test("david@school.co.il"); // true emailPattern.test("not-an-email"); // false emailPattern.test("@missing.com"); // false

עכשיו נשכתב את הבדיקות מסעיף 2 בצורה תמציתית:

// שם משתמש: מתחיל באות, לפחות 3 תווים /^[a-zA-Z].{2,}$/.test(username) // סיסמה: 6-12 תווים /^.{6,12}$/.test(password) // סיסמה: חייבת ספרה /\d/.test(password) // סיסמה: חייבת תו מיוחד /[!@#$%^&*]/.test(password) // אימייל: פורמט בסיסי /^[\w.-]+@[\w.-]+\.\w{2,}$/.test(email) // טלפון: 10 ספרות, מתחיל ב-05 /^05\d{8}$/.test(phone)
✔️ שימו לב להבדל

עם RegEx — כל בדיקה היא שורה אחת במקום פונקציה שלמה עם לולאות ותנאים. זה קוד הרבה יותר קריא ותחזוקתי.

4. שימוש ב-onsubmit

אז יש לנו חוקים ויש לנו RegEx. עכשיו צריך לחבר את הכל: כשהמשתמש לוחץ "שלח" — לבדוק את כל השדות, ואם משהו לא תקין — לעצור את שליחת הטופס ולהציג הודעות שגיאה.

העיקרון: return false עוצר שליחה

נוסיף לטופס את המאפיין onsubmit שמפעיל פונקציית בדיקה:

<form onsubmit="return validate()"> ... <button type="submit">שלח</button> </form>

אם הפונקציה validate() מחזירה false — הטופס לא יישלח. אם היא מחזירה true — הטופס יישלח כרגיל.

הצגת שגיאות ליד כל שדה

לכל שדה בטופס נוסיף <span class="error"> עם הודעת שגיאה, שכברירת מחדל מוסתר (display:none). כשהבדיקה נכשלת — נחשוף אותו:

<div class="form-group"> <label>שם משתמש:</label> <input type="text" id="username"> <span class="error" id="usernameErr"> חייב להתחיל באות, לפחות 3 תווים </span> </div>

פונקציית ולידציה מלאה

הנה דוגמה מלאה שבודקת שם משתמש, אימייל וסיסמה:

function validate() { let valid = true; // --- שם משתמש --- let user = document.getElementById("username").value; if (!/^[a-zA-Z].{2,}$/.test(user)) { document.getElementById("usernameErr").style.display = "block"; document.getElementById("username").className = "invalid"; valid = false; } else { document.getElementById("usernameErr").style.display = "none"; document.getElementById("username").className = "valid"; } // --- אימייל --- let email = document.getElementById("email").value; if (!/^[\w.-]+@[\w.-]+\.\w{2,}$/.test(email)) { document.getElementById("emailErr").style.display = "block"; document.getElementById("email").className = "invalid"; valid = false; } else { document.getElementById("emailErr").style.display = "none"; document.getElementById("email").className = "valid"; } // --- סיסמה --- let pass = document.getElementById("password").value; if (!/^.{6,12}$/.test(pass) || !/\d/.test(pass) || !/[!@#$%^&*]/.test(pass)) { document.getElementById("passErr").style.display = "block"; document.getElementById("password").className = "invalid"; valid = false; } else { document.getElementById("passErr").style.display = "none"; document.getElementById("password").className = "valid"; } return valid; // true = שלח, false = עצור }
💡 דפוס חשוב

שימו לב לדפוס: מתחילים עם let valid = true, ובכל בדיקה שנכשלת מסמנים valid = false. בסוף מחזירים את valid. ככה כל השגיאות מוצגות בו-זמנית, ולא רק הראשונה.

משוב ויזואלי: גבולות ירוקים ואדומים

שדה תקין מקבל גבול ירוק (class="valid"), שדה שגוי מקבל גבול אדום (class="invalid"). ב-CSS:

input.valid { border-color: green; } input.invalid { border-color: red; } .error { color: red; font-size: 14px; display: none; }

שילוב של גבול צבעוני + הודעת טקסט ברורה נותן למשתמש את המשוב הכי טוב: הוא רואה מה לא תקין ולמה.

📝 תרגיל: בניית דף הרשמה מלא

בנו טופס הרשמה מלא בעורך הקוד למעלה (מחקו את הקוד הקיים והתחילו מאפס, או הרחיבו אותו). הטופס צריך לכלול את כל השדות הבאים:

  1. שם מלא — לפחות 2 תווים
  2. אימייל — פורמט תקין (user@domain.com)
  3. טלפון — 10 ספרות, מתחיל ב-05
  4. סיסמה — 6-12 תווים, חייבת ספרה + תו מיוחד
  5. אישור סיסמה — חייב להיות זהה לסיסמה
  6. תאריך לידה — חייב להיות בן 10 לפחות (חשבו: להשוות תאריכים)

לכל שדה:

  • הוסיפו הודעת שגיאה ייחודית שמסבירה בדיוק מה לא תקין
  • סמנו את השדה בגבול אדום (invalid) או ירוק (valid)
  • הציגו הודעת הצלחה אם כל השדות תקינים

בונוס: הוסיפו "מד חוזק סיסמה" (password strength meter) — פס צבעוני שמשתנה מאדום (חלש) לירוק (חזק) בזמן ההקלדה.
