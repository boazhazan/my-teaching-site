Connected Method, ExecuteNonQuery, asp-route

1. סקירת השיטה המקושרת

בשיטה המקושרת (Connected), פותחים חיבור, מריצים פקודה ישירות, וסוגרים — בלי DataSet.

שימושי בעיקר לפעולות כתיבה: INSERT, UPDATE, DELETE — כשלא צריך לקרוא נתונים חזרה.

2. מחיקה (DELETE) עם ExecuteNonQuery

public void OnPostDelete(int id) { string conStr = "..."; using (SqlConnection con = new SqlConnection(conStr)) { string query = "DELETE FROM Students WHERE Id = @id"; SqlCommand cmd = new SqlCommand(query, con); cmd.Parameters.AddWithValue("@id", id); con.Open(); cmd.ExecuteNonQuery(); con.Close(); } }

ב-HTML, העברת ה-ID מהטבלה לכפתור:

<!-- asp-route-id מעביר את ה-ID --> <form method="post" asp-page-handler="Delete" asp-route-id="@row["Id"]"> <button onclick="return confirm('למחוק?')">מחק</button> </form>

4. עדכון (UPDATE)

זרימה דו-שלבית: OnGet טוען את הנתונים הנוכחיים, OnPost שומר את השינויים.

// OnGet — טוען נתונים קיימים לטופס public void OnGet(int id) { string query = "SELECT * FROM Students WHERE Id = @id"; // Fill form fields from DataTable... } // OnPost — שומר את השינויים public void OnPost() { string query = "UPDATE Students SET Name=@name, Grade=@grade WHERE Id=@id"; using (SqlConnection con = new SqlConnection(conStr)) { SqlCommand cmd = new SqlCommand(query, con); cmd.Parameters.AddWithValue("@name", Name); cmd.Parameters.AddWithValue("@grade", Grade); cmd.Parameters.AddWithValue("@id", StudentId); con.Open(); cmd.ExecuteNonQuery(); } }

5. טיפול בשגיאות — try/catch

try { con.Open(); cmd.ExecuteNonQuery(); Message = "הנתונים עודכנו בהצלחה!"; } catch (SqlException ex) { if (ex.Number == 2627) // UNIQUE constraint ErrorMessage = "ערך כפול — כבר קיים!"; else ErrorMessage = "שגיאה: " + ex.Message; } finally { con.Close(); }
using = finally אוטומטי

כשמשתמשים ב-using, החיבור נסגר אוטומטית גם אם יש שגיאה — לא צריך finally.
