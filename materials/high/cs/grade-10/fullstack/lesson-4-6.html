שיעור 4.6 — ExecuteScalar, DataReader ו-SQL Injection

שיטות שאילתה נוספות ואבטחת מידע

1. ExecuteScalar — ערך בודד

מחזיר את הערך הראשון בשורה הראשונה. מתאים ל-COUNT, MAX, MIN, login check.

// כמה תלמידים יש? string query = "SELECT COUNT(*) FROM Students"; SqlCommand cmd = new SqlCommand(query, con); con.Open(); int count = (int)cmd.ExecuteScalar(); con.Close(); // count = 42

שימוש ל-Login

string query = "SELECT COUNT(*) FROM Users WHERE Username=@u AND Password=@p"; cmd.Parameters.AddWithValue("@u", Username); cmd.Parameters.AddWithValue("@p", Password); int found = (int)cmd.ExecuteScalar(); if (found > 0) // Login success! else // Login failed

2. DataReader — קריאה קדימה בלבד

DataReader קורא שורות אחת-אחת, בלי לטעון הכל לזיכרון. מהיר ויעיל!

SqlDataReader reader = cmd.ExecuteReader(); while (reader.Read()) { string name = reader.GetString(0); // עמודה 0 int grade = reader.GetInt32(1); // עמודה 1 Console.WriteLine($"{name}: {grade}"); } reader.Close();
Forward-only

DataReader יכול רק להתקדם — אין אפשרות לחזור לשורה קודמת. אם צריך לנווט חופשי, השתמשו ב-DataTable.

3. SQL Injection — התקפת הזרקה

סכנה!

SQL Injection היא אחת ההתקפות הנפוצות ביותר באינטרנט. כל מפתח חייב להכיר אותה!

כשמשרשרים קלט משתמש ישירות לשאילתה:

// ❌ מסוכן!! string query = "SELECT * FROM Users WHERE Username='" + username + "'"; // אם המשתמש מקליד: ' OR 1=1 -- // השאילתה הופכת ל: SELECT * FROM Users WHERE Username='' OR 1=1 --' // 1=1 תמיד TRUE → מחזיר את כל המשתמשים! // -- הופך את השאר להערה

4. SqlParameter — הפתרון

// ✓ בטוח! string query = "SELECT * FROM Users WHERE Username=@user AND Password=@pass"; SqlCommand cmd = new SqlCommand(query, con); cmd.Parameters.AddWithValue("@user", username); cmd.Parameters.AddWithValue("@pass", password);
כלל הזהב

לעולם אל תשרשרו קלט משתמש לתוך SQL! תמיד השתמשו ב-SqlParameter. הפרמטר מטופל כערך, לא כקוד SQL.
