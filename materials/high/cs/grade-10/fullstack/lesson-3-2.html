שיעור 3.2 — תחביר Razor

העברת נתונים בין cshtml ו-cshtml.cs

1. הצגת ערכים עם @

סימן @ ב-Razor אומר לשרת: "הכנס כאן ערך מ-C#". זה הגשר בין הקוד לבין ה-HTML.

@{ string name = "בועז"; int age = 30; } <h1>שלום @name!</h1> <p>גיל: @age</p> <p>תאריך: @DateTime.Now.ToString("dd/MM/yyyy")</p>

@Html.Raw()

כברירת מחדל, Razor מבצע escape לתגיות HTML (מציג אותן כטקסט). כדי להכניס HTML אמיתי:

@{ string html = "<b>טקסט מודגש</b>"; } <p>@html</p> <!-- מציג: <b>טקסט מודגש</b> --> <p>@Html.Raw(html)</p> <!-- מציג: טקסט מודגש -->

2. Model Properties

הדרך המומלצת להעביר נתונים: הגדרת Properties ב-.cshtml.cs וגישה דרך @Model.

Index.cshtml.cs
public class IndexModel : PageModel { public string Title { get; set; } public int Year { get; set; } public void OnGet() { Title = "דף הבית"; Year = DateTime.Now.Year; } }
Index.cshtml
<h1>@Model.Title</h1> <p>© @Model.Year</p>

3. ViewData

ViewData הוא מילון (Dictionary) להעברת נתונים נוספים. פחות בטוח מ-Model כי אין Type Safety.

// בקוד (cshtml.cs) public void OnGet() { ViewData["Message"] = "ברוכים הבאים!"; ViewData["Count"] = 42; } // בדף (cshtml) <p>@ViewData["Message"]</p> <p>יש @ViewData["Count"] פריטים</p>
ViewData vs Model

Model = strongly typed (בטוח, עם IntelliSense). ViewData = dictionary (צריך casting, מתאפס כל request). העדיפו Model!
