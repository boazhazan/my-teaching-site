1. מודל Client-Server

עד עכשיו בנינו דפי HTML שנפתחים ישירות בדפדפן — קבצים סטטיים שלא משתנים. אבל רוב האתרים שאנחנו מכירים (יוטיוב, אינסטגרם, גוגל) עובדים אחרת לגמרי: התוכן נבנה דינמית בשרת לפי הבקשה של המשתמש.

השחקנים במשחק

  • לקוח (Client) — הדפדפן שלכם (Chrome, Edge, Firefox). הוא שולח בקשות (Requests) ומציג את התוצאה.
  • שרת (Server) — מחשב חזק שרץ 24/7 ומחכה לבקשות. כשמגיעה בקשה, הוא מעבד אותה ומחזיר תשובה (Response) — בדרך כלל HTML, CSS ו-JS.
  • פרוטוקול HTTP — "שפת התקשורת" בין הלקוח לשרת. כל בקשה ותשובה עוברות דרך HTTP (או HTTPS מוצפן).

איך זה עובד?

  1. המשתמש מקליד כתובת URL בדפדפן (למשל www.example.com/about)
  2. הדפדפן שולח HTTP Request לשרת
  3. השרת מעבד את הבקשה (מריץ קוד, קורא ממסד נתונים...)
  4. השרת בונה דף HTML עם הנתונים ומחזיר HTTP Response
  5. הדפדפן מקבל את ה-HTML ומציג אותו על המסך

דוגמה מהחיים:

כשאתם גולשים ליוטיוב, הדפדפן שולח בקשה לשרתים של גוגל. השרת בודק מי אתם, מה ההמלצות שלכם, ובונה דף HTML מותאם אישית — עם סרטונים שונים לכל משתמש. זה ההבדל מ-HTML סטטי: התוכן נבנה תוך כדי ולא קבוע מראש.

סוגי בקשות HTTP נפוצים

  • GET — "תן לי מידע" (כניסה לדף, חיפוש). הנפוצה ביותר.
  • POST — "קבל ממני מידע" (שליחת טופס, העלאת קובץ).

2. מה זה Razor Pages?

Razor Pages היא טכנולוגיה של Microsoft לבניית אתרים דינמיים, חלק מ-ASP.NET Core — הפלטפורמה המודרנית של מיקרוסופט לפיתוח Web.

הרעיון המרכזי: כל דף באתר מורכב משני קבצים שעובדים יחד:

קובץ תפקיד שפה
.cshtml העיצוב — HTML עם קוד C# מוטמע בעזרת @ HTML + C# (Razor)
.cshtml.cs הלוגיקה — Code-behind שמריץ את הפעולות C#

דוגמה: קובץ Index.cshtml (העיצוב)

@page @model IndexModel <h1>@Model.Message</h1> <p>היום: @DateTime.Now.ToShortDateString()</p>

דוגמה: קובץ Index.cshtml.cs (הלוגיקה)

public class IndexModel : PageModel { public string Message { get; set; } public void OnGet() { Message = "ברוכים הבאים לאתר!"; } }
שימו לב! C# רצה בשרת, לא בדפדפן

הקוד שכותבים ב-Razor Pages רץ בצד השרת. הדפדפן מקבל רק HTML רגיל — הוא לא רואה את קוד ה-C# בכלל. זה שונה מ-JavaScript שרצה בתוך הדפדפן (צד הלקוח).

ההבדל: HTML סטטי מול Razor Pages

HTML סטטי Razor Pages
תוכן קבוע — אותו דבר לכולם דינמי — נבנה בשרת לפי נתונים
שפה HTML + CSS + JS HTML + CSS + JS + C#
מסד נתונים אין גישה יש גישה דרך C#
דוגמה דף "אודות" פשוט פרופיל משתמש, חנות, פורום
הסימן @ — הכוכב של Razor

בקובץ .cshtml, הסימן @ מסמן מעבר מ-HTML לקוד C#.
@Model.Name — הצגת ערך של משתנה.
@DateTime.Now — הרצת פקודת C# ישירות.
@{ ... } — בלוק קוד C# שלם.

3. ניתוב (Routing)

ניתוב (Routing) זה הקשר בין ה-URL שהמשתמש מקליד לבין הקובץ שהשרת מפעיל. ב-Razor Pages הניתוב הוא אוטומטי לפי מיקום הקובץ בפרויקט — לא צריך להגדיר כלום ידנית!

הכלל פשוט

הנתיב של הקובץ בתוך תיקיית Pages הופך ל-URL:

Pages/ ├── Index.cshtml → www.site.com/ ├── About.cshtml → www.site.com/About ├── Contact.cshtml → www.site.com/Contact └── Admin/ ├── Dashboard.cshtml → www.site.com/Admin/Dashboard └── Users.cshtml → www.site.com/Admin/Users
מיקום הקובץ URL הערה
Pages/Index.cshtml / דף הבית — Index הוא ברירת מחדל
Pages/About.cshtml /About שם הקובץ = נתיב ה-URL
Pages/Admin/Dashboard.cshtml /Admin/Dashboard תת-תיקייה = נתיב מקונן
למה זה נוח?

בטכנולוגיות אחרות צריך להגדיר טבלת ניתוב מסובכת. ב-Razor Pages פשוט יוצרים קובץ במקום הנכון, וה-URL נוצר אוטומטית. רוצים דף חדש? יוצרים קובץ חדש בתיקייה.

4. Handlers — טיפול בבקשות

בכל דף Razor Pages יש Handlers — פונקציות שמטפלות בסוגי בקשות שונים. שני ה-Handlers הנפוצים:

  • OnGet() — מופעל כשהמשתמש נכנס לדף (בקשת GET). כאן מכינים את הנתונים לתצוגה.
  • OnPost() — מופעל כשנשלח טופס (בקשת POST). כאן מעבדים את הנתונים שהמשתמש שלח.

דוגמה מלאה: דף ברכה

קובץ Greeting.cshtml.cs (הלוגיקה):

public class GreetingModel : PageModel { public string Message { get; set; } public int VisitCount { get; set; } public void OnGet() { Message = "ברוכים הבאים!"; VisitCount = 42; // בפועל נקרא ממסד נתונים } public void OnPost() { // עיבוד נתוני הטופס Message = "הטופס נשלח בהצלחה!"; } }

קובץ Greeting.cshtml (העיצוב):

@page @model GreetingModel <h1>@Model.Message</h1> <p>מספר ביקורים: @Model.VisitCount</p> <form method="post"> <input type="text" name="userName" /> <button type="submit">שלח</button> </form>

מה קורה כשמשתמש נכנס לדף?

  1. המשתמש מקליד www.site.com/Greeting
  2. השרת מזהה שזו בקשת GET ומפעיל את OnGet()
  3. OnGet() מגדיר Message = "ברוכים הבאים!"
  4. השרת לוקח את קובץ ה-.cshtml, מחליף את @Model.Message בערך האמיתי, ובונה HTML
  5. ה-HTML נשלח ללקוח — הדפדפן מציג "ברוכים הבאים!"

מה קורה כשנשלח טופס?

  1. המשתמש ממלא את הטופס ולוחץ "שלח"
  2. הדפדפן שולח בקשת POST עם הנתונים
  3. השרת מפעיל את OnPost()
  4. הלוגיקה מעבדת את הנתונים ומעדכנת את Message
  5. הדף נבנה מחדש ונשלח — עכשיו מציג "הטופס נשלח בהצלחה!"
PageModel — מחלקת הבסיס

כל מחלקת Code-behind יורשת מ-PageModel. המאפיינים (Properties) שמגדירים במחלקה (כמו Message, VisitCount) זמינים בקובץ ה-.cshtml דרך @Model.PropertyName.
