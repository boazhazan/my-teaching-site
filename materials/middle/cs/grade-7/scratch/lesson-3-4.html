יחידה 3: משתנים ותנאים

משתנה מונה ומשתנה צובר

ספירה, סכימה, משחק פולשי החלל ודפוסי צבירה

1. מהם דפוסי מונה וצובר?

בשיעורים הקודמים למדנו מה זה משתנה ואיך לשנות את הערך שלו. היום נלמד על שני דפוסים חשובים שחוזרים שוב ושוב בתכנות — ובמשחקים:

  • מונה (Counter) — משתנה שסופר כמה פעמים משהו קרה. עולה ב-1 בכל פעם
  • צובר (Accumulator) — משתנה שצובר (סוכם) ערכים. מוסיף כמויות שונות בכל פעם

שני הדפוסים עובדים באותו עיקרון:

  1. אתחול — קובעים ערך התחלתי (בדרך כלל 0)
  2. לולאה / אירוע — כל פעם שמשהו קורה...
  3. עדכון — משנים את המשתנה

ההבדל? מונה תמיד עולה ב-1 (כמות קבועה), ואילו צובר מוסיף כמויות שמשתנות — לפעמים 10, לפעמים 50, לפעמים את מה שהמשתמש הקליד.

💡 למה זה חשוב?

כמעט כל משחק או תוכנית משתמשים בדפוסים האלה. ניקוד, חיים, כסף, ספירת זמן, ממוצע ציונים — כולם מבוססים על מונה או צובר. ברגע שתזהו את הדפוס, תוכלו ליישם אותו בכל מצב!

2. משתנה מונה (Counter)

מונה הוא המקרה הפשוט ביותר — אנחנו רוצים לספור כמה פעמים משהו קרה. הדפוס תמיד אותו דבר:

  1. אתחול: קבע את המונה ל-0
  2. בכל אירוע: שנה את המונה ב-1

דוגמאות לשימושים במונה

  • ספירת לחיצות — כל לחיצה על הדמות = +1
  • ספירת חיים — התחלה ב-3, כל פגיעה = −1
  • ספירת סיבובים — כל סיבוב שלם = +1
  • ספירת אויבים — כל אויב שהובס = +1

דוגמה: מונה לחיצות

בואו נבנה תוכנית פשוטה שסופרת כמה פעמים לחצנו על הדמות:

when 🟢 clicked
set מונה to 0
when this sprite clicked
change מונה by 1
say מונה for 1 seconds

מה קורה כאן?

  • כשלוחצים על הדגל הירוק — המונה מתאפס ל-0
  • כל פעם שלוחצים על הדמות — המונה עולה ב-1 והדמות מציגה את הערך
  • לחיצה ראשונה: 0 → 1. לחיצה שנייה: 1 → 2. וכן הלאה
✅ שימו לב לאתחול!

בלי הבלוק "קבע מונה ל-0" בתחילת התוכנית, המונה ימשיך מאיפה שהוא הפסיק בהפעלה הקודמת. תמיד תזכרו לאתחל את המונה כשהתוכנית מתחילה!

3. משתנה צובר (Accumulator)

צובר דומה למונה, אבל במקום להוסיף תמיד 1 — אנחנו מוסיפים כמויות שונות בכל פעם. הדפוס:

  1. אתחול: קבע את הסכום ל-0
  2. בכל אירוע: שנה את הסכום ב-(הערך הנוכחי)

דוגמאות לשימושים בצובר

  • ניקוד כולל — אויב קטן = +10 נקודות, אויב גדול = +50 נקודות
  • סכום כסף — קונים ומוכרים בסכומים שונים
  • סכום מספרים — המשתמש מקליד מספרים ואנחנו סוכמים

דוגמה: מחשבון סכימה

תוכנית שמבקשת מהמשתמש להכניס 5 מספרים ומציגה את הסכום שלהם:

when 🟢 clicked
set סכום to 0
repeat 5
ask הכנס מספר and wait
change סכום by answer
say join [הסכום הוא: ] [סכום] for 3 seconds

מה קורה כאן?

  • בהתחלה, הסכום מתאפס ל-0
  • הלולאה רצה 5 פעמים — כל פעם שואלת מספר ומוסיפה אותו לסכום
  • לדוגמה: אם הוכנסו 3, 7, 2, 10, 8 — הסכום יהיה 30
  • בסוף, הדמות מציגה את הסכום הכולל
⚠️ שימו לב להבדל!

במונה כתבנו "שנה ב-1" (מספר קבוע). בצובר כתבנו "שנה ב-(תשובה)" (ערך שמשתנה). זה ההבדל המהותי בין שני הדפוסים!

4. ההבדל בין מונה לצובר

עכשיו שראינו את שני הדפוסים, בואו נשווה ביניהם:

מונה (Counter) צובר (Accumulator)
מטרה לספור כמה פעמים משהו קרה לצבור / לסכום ערכים
שינוי בכל צעד תמיד +1 (או −1) כמות משתנה (+10, +50, +תשובה...)
ערך התחלתי בדרך כלל 0 בדרך כלל 0
דוגמאות חיים, לחיצות, סיבובים ניקוד, סכום כסף, סכום מספרים
הבלוק שנה [מונה] ב-1 שנה [סכום] ב-(ערך)
💡 תובנה חשובה

מונה הוא בעצם מקרה פרטי של צובר! מונה הוא צובר שתמיד מוסיף 1. אם תבינו את דפוס הצובר — תבינו גם את המונה.

5. משחק פולשי החלל (Space Invaders)

בואו נראה איך שני הדפוסים — מונה וצובר — עובדים יחד באותו משחק! במשחק פולשי החלל יש לנו:

  • חיים (מונה יורד) — מתחילים עם 3 חיים, כל פגיעה מורידה 1
  • ניקוד (צובר) — אויב קטן = +10 נקודות, אויב גדול = +50 נקודות
  • זמן (מונה עולה) — סופר כמה שניות שרדנו

קוד ההתחלה

when 🟢 clicked
set ניקוד to 0
set חיים to 3

כאשר פוגעים באויב קטן (צובר — +10)

change ניקוד by 10

כאשר פוגעים באויב גדול (צובר — +50)

change ניקוד by 50

כאשר השחקן נפגע (מונה — −1)

change חיים by -1
if (חיים) = (0) then
say !Game Over for 2 seconds
stop all

שימו לב למה שקורה כאן:

  • ניקוד הוא צובר — מוסיפים לו כמויות שונות (10 או 50)
  • חיים הוא מונה יורד — מורידים ממנו 1 בכל פגיעה
  • כשהחיים מגיעים ל-0 — המשחק נגמר!
✅ דפוסים בפעולה

במשחק אחד יכולים לחיות כמה מונים וכמה צוברים יחד! המפתח הוא לזהות לכל משתנה: האם הוא סופר אירועים (מונה) או צובר ערכים (צובר)?

6. ממוצע — שילוב מונה וצובר

אחד השימושים הכי מגניבים הוא חישוב ממוצע. כדי לחשב ממוצע, צריכים גם מונה וגם צובר:

  • מונה — כמה מספרים הוכנסו (ספירה)
  • צובר — הסכום של כל המספרים
  • ממוצע = סכום / כמות

דוגמה: מחשבון ממוצע

when 🟢 clicked
set סכום to 0
set כמות to 0
repeat 5
ask הכנס מספר and wait
change סכום by answer
change כמות by 1
set ממוצע to (סכום) / (כמות)
say join [הממוצע הוא: ] [ממוצע] for 3 seconds

בואו נעקוב אחרי דוגמה. נניח שהמשתמש מכניס: 10, 20, 30, 40, 50

שלב מספר שהוכנס סכום (צובר) כמות (מונה)

בסוף: ממוצע = 150 / 5 = 30

💡 שימו לב לשילוב

בתוך הלולאה יש שתי פעולות עדכון — אחת לצובר ("שנה סכום ב-תשובה") ואחת למונה ("שנה כמות ב-1"). זה מה שמאפשר לנו לחשב ממוצע אחר כך!

📝 תרגיל: בנו בסקראץ'

פתחו את סקראץ' ונסו לבנות את התוכניות הבאות:

  1. מונה לחיצות — צרו משתנה "מונה" שמתחיל ב-0. כל פעם שלוחצים על הדמות, המונה עולה ב-1 והערך מוצג על הבמה. הוסיפו גם הודעה ויזואלית של הדמות עם הערך הנוכחי
  2. מחשבון סכימה — צרו תוכנית ששואלת מהמשתמש 5 מספרים (אחד אחרי השני) וסוכמת אותם. בסוף, הדמות מציגה הודעה: "הסכום הוא: ___"
  3. מחשבון ממוצע — שפרו את תוכנית הסכימה כך שהיא גם מחשבת ומציגה את הממוצע של 5 המספרים. תצטרכו גם מונה וגם צובר!

בונוס: צרו מערכת ניקוד פשוטה למשחק פולשי החלל — 3 חיים, ניקוד שעולה ב-10 או 50 לפי סוג האויב, והודעת Game Over כשהחיים מגיעים ל-0!
