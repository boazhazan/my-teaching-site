רשימות ומחרוזות — סיכום

שילוב רשימות ומחרוזות, חיפוש, סינון ופרויקט מסכם

1. חזרה — מחרוזות ורשימות

לפני שנמשיך הלאה, בואו נעשה סדר. למדנו שני מבני נתונים חשובים בסקראץ' — מחרוזות ורשימות. הם דומים בכמה דברים, אבל שונים מאוד בשימוש.

מחרוזת (String)

  • רצף של תווים (אותיות, מספרים, סימנים)
  • שרשור — חיבור שתי מחרוזות עם הבלוק join
  • אות לפי מיקום — letter (N) of
  • אורך — length of
  • מכיל — contains

רשימה (List)

  • אוסף מסודר של פריטים (מספרים, טקסט, כל דבר!)
  • הוספה — add לסוף הרשימה
  • מחיקה — delete לפי מיקום
  • גישה לפי אינדקס — item (N) of
  • אורך — length of
  • מכיל — contains
💡 מה המשותף?

שני מבני הנתונים מתחילים מאינדקס 1 (ולא 0 כמו בשפות אחרות). לשניהם יש בלוקי "אורך" ו"מכיל". ההבדל המרכזי: מחרוזת מחזיקה תווים בלבד, בעוד רשימה יכולה להחזיק כל ערך — מספרים, מילים שלמות, ואפילו משפטים.

2. שילוב מחרוזות ורשימות

הכוח האמיתי מגיע כשמשלבים את שני מבני הנתונים יחד. הנה כמה דפוסים נפוצים:

פירוק מחרוזת לאותיות ברשימה

נניח שיש לנו מילה ואנחנו רוצים לשמור כל אות שלה בנפרד ברשימה. זה שימושי למשחקי מילים, הצפנה, ועוד:

set מילה to שלום
delete all of אותיות
set i to 1
repeat length of מילה
add letter i of מילה to אותיות
change i by 1

אחרי הרצת הקוד, הרשימה אותיות תכיל: ש, ל, ו, ם — כל אות בפריט נפרד!

בניית משפט מרשימה

הכיוון ההפוך — יש לנו רשימת מילים ואנחנו רוצים לחבר אותן למשפט אחד:

set משפט to
set i to 1
repeat length of מילים
set משפט to join משפט join item i of מילים
change i by 1

אם הרשימה מילים מכילה: "אני", "אוהב", "סקראץ'" — התוצאה תהיה המחרוזת: אני אוהב סקראץ'

⚠️ שימו לב לרווחים!

כשמחברים מילים למשפט, אל תשכחו להוסיף רווח בין המילים. בלי הרווח תקבלו "אניאוהבסקראץ'" — מחרוזת אחת ארוכה בלי הפרדה.

3. חיפוש ברשימה

אחת הפעולות הכי נפוצות בתכנות היא חיפוש — מציאת ערך מסוים בתוך אוסף נתונים. בסקראץ' יש לנו שתי גישות:

גישה פשוטה: "מכיל?"

הבלוק contains עונה כן/לא — האם הפריט קיים ברשימה. מושלם כשרוצים רק לדעת אם משהו קיים:

if שמות contains תשובה? then
say השם קיים ברשימה!

חיפוש עם מציאת מיקום (אינדקס)

לפעמים לא מספיק לדעת אם — אנחנו רוצים לדעת איפה. לדוגמה: "באיזה מקום ברשימה נמצא השם?" לזה צריך חיפוש ליניארי — לעבור פריט-פריט ולבדוק:

ask חפש שם: and wait
set חיפוש to answer
set נמצא to 0
set i to 1
repeat length of שמות
if item i of שמות = חיפוש then
set נמצא to i
change i by 1
if נמצא > 0 then
say join נמצא במקום נמצא
else
say לא נמצא
💡 למה מאתחלים את "נמצא" ל-0?

כי אין פריט במיקום 0 ברשימה. אם אחרי הלולאה הערך עדיין 0 — סימן שהפריט לא נמצא. אם הערך גדול מ-0 — זה המיקום שלו ברשימה!

4. סינון רשימה

סינון הוא פעולה שבה יוצרים רשימה חדשה שמכילה רק את הפריטים שעומדים בתנאי מסוים. הדפוס: עוברים על הרשימה המקורית, בודקים כל פריט, ואם הוא עומד בתנאי — מוסיפים אותו לרשימה החדשה.

דוגמה: סינון ציונים מעל 80

נניח שיש לנו רשימת ציונים ואנחנו רוצים ליצור רשימה חדשה ציונים-גבוהים עם כל הציונים שמעל 80:

delete all of ציונים-גבוהים
set i to 1
repeat length of ציונים
if item i of ציונים > 80 then
add item i of ציונים to ציונים-גבוהים
change i by 1

דוגמה: סינון שמות שמתחילים ב-"א"

כאן משלבים מחרוזת ורשימה! בודקים את האות הראשונה של כל פריט ברשימה:

delete all of שמות-א
set i to 1
repeat length of שמות
if letter 1 of item i of שמות = א then
add item i of שמות to שמות-א
change i by 1
✅ דפוס הסינון — שמרו אותו!

הדפוס תמיד זהה: (1) ניקוי רשימת היעד, (2) לולאה על הרשימה המקורית, (3) תנאי, (4) הוספה אם עומד בתנאי. רק התנאי משתנה!

5. מיון בועות (Bubble Sort) — מושג מתקדם

נניח שיש לנו רשימת מספרים ואנחנו רוצים לסדר אותה מהקטן לגדול. איך עושים את זה? נשתמש באלגוריתם מפורסם בשם מיון בועות (Bubble Sort).

איך זה עובד?

  1. עוברים על הרשימה ומשווים כל שני פריטים שכנים
  2. אם השמאלי (הקודם) גדול מהימני (הבא) — מחליפים ביניהם
  3. חוזרים על התהליך שוב ושוב, עד שאין יותר החלפות
  4. כל סיבוב "מבעבע" את המספר הגדול ביותר למקום הנכון שלו

הנה המימוש בסקראץ':

set הוחלף to 1
repeat until הוחלף = 0
set הוחלף to 0
set i to 1
repeat length of מספרים - 1
if item i of מספרים > item i + 1 of מספרים then
set זמני to item i of מספרים
replace item i of מספרים with item i + 1 of מספרים
replace item i + 1 of מספרים with זמני
set הוחלף to 1
change i by 1
⚠️ למה צריך משתנה "זמני"?

כדי להחליף בין שני ערכים, חייבים לשמור את אחד מהם בצד (במשתנה זמני). אחרת, כשנדרוס את הערך הראשון — נאבד אותו לתמיד! זה כמו להחליף תכולה בין שני כוסות — צריך כוס שלישית ריקה.

💡 עובדה מעניינת

מיון בועות הוא אחד האלגוריתמים הראשונים שלומדים במדעי המחשב. הוא לא הכי יעיל (יש אלגוריתמים מהירים יותר), אבל הוא הכי קל להבנה — ולכן מושלם ללמידה. בקורסים מתקדמים תלמדו גם Merge Sort, Quick Sort ועוד!

6. פרויקט מסכם: חידון טריוויה עם רשימות

בואו נבנה משחק שמשלב את כל מה שלמדנו — רשימות, מחרוזות, חיפוש, ולוגיקה. הרעיון: חידון טריוויה שבו השאלות והתשובות מאוחסנות ברשימות מקבילות.

מה זה "רשימות מקבילות"?

שתי רשימות שבהן אותו אינדקס מתייחס לאותו דבר. לדוגמה:

  • רשימת שאלות: ["בירת ישראל?", "כמה רגליים לעכביש?", "באיזה כוכב אנחנו חיים?"]
  • רשימת תשובות: ["ירושלים", "8", "כדור הארץ"]

שאלה 1 → תשובה 1, שאלה 2 → תשובה 2, וכן הלאה. אותו אינדקס!

מבנה המשחק

when 🏴 clicked
set ניקוד to 0
set סיבוב to 0
repeat 5
change סיבוב by 1
set מספר-שאלה to pick random 1 to length of שאלות
ask item מספר-שאלה of שאלות and wait
if answer = item מספר-שאלה of תשובות then
say נכון! 🎉 for 2 seconds
change ניקוד by 1
else
say join לא נכון. התשובה: item מספר-שאלה of תשובות for 3 seconds
say join סיימת! הניקוד שלך: ניקוד
✅ למה רשימות מקבילות זה כוח?

עם אינדקס אחד אפשר לגשת לכל המידע הקשור. זה דפוס שמופיע בכל שפת תכנות — לא רק בסקראץ'. בשפות מתקדמות יש "מילון" (Dictionary) שעושה את זה בצורה אלגנטית יותר, אבל הרעיון זהה!

מה הלאה?

  • בנו משחק שלם! — קחו את כל הכלים שלמדתם ובנו פרויקט גדול: משחק הרפתקאות, פלטפורמר, חידון, או כל דבר שתדמיינו

✅ הדבר הכי חשוב שלמדתם

מעבר לסקראץ' עצמו, למדתם לחשוב כמו מתכנתים: פירוק בעיות (Decomposition), זיהוי דפוסים (Pattern Recognition), הפשטה (Abstraction), ותכנון אלגוריתמי. הכישורים האלה רלוונטיים לכל שפת תכנות — ולכל תחום בחיים!

📝 תרגיל מסכם — פרויקט לבחירה

בחרו אחד (או יותר!) מהפרויקטים הבאים ובנו אותו בסקראץ'. השתמשו בכל מה שלמדתם!

  1. חידון מילים — רשימת מילים בעברית + רשימת תרגומים באנגלית (רשימות מקבילות). הגריל מילה אקראית, בקש תרגום מהשחקן, בדוק תשובה, עקוב אחרי ניקוד. בסוף — הצג כמה תשובות נכונות מתוך הכלל
  2. טבלת שיאים — רשימת שמות + רשימת ניקודים. הוסף שחקנים חדשים, מצא את הניקוד הגבוה ביותר (מי האלוף?), מיין מהגבוה לנמוך (שימוש במיון בועות!)
  3. משחק "עצם או חיה?" — שתי רשימות: חיות ועצמים. המחשב בוחר פריט אקראי מרשימה אקראית. השחקן מנחש מאיזו קטגוריה הפריט. השתמשו ב-contains לבדוק!
  4. מגדיר מילון — רשימת מילים + רשימת הגדרות. המשתמש מקליד מילה, התוכנית מחפשת אותה (חיפוש ליניארי) ומציגה את ההגדרה. אם לא נמצאה — הצע להוסיף!

בונוס — המצאה חופשית: המציאו משחק או כלי משלכם שמשתמש ברשימות, מחרוזות, My Blocks, ולפחות אלגוריתם אחד (חיפוש/סינון/מיון). זה הפרויקט שלכם — תנו לדמיון לרוץ! 🚀
