שיעור 4.3 — סינון, מיון ו-asp-page-handler

WHERE LIKE, ORDER BY, מספר טפסים בדף אחד

1. מספר טפסים בדף — asp-page-handler

מה קורה כשיש שני טפסים באותו דף — גם סינון וגם מיון? איך השרת יודע מאיזה טופס הגיעה הבקשה?

הפתרון: asp-page-handler שמנתב כל טופס ל-handler נפרד.

<!-- טופס סינון → OnPostFilter() --> <form method="post" asp-page-handler="Filter"> <input asp-for="SearchText" placeholder="חפש שם..." /> <button type="submit">סנן</button> </form> <!-- טופס מיון → OnPostSort() --> <form method="post" asp-page-handler="Sort"> <select asp-for="SortBy"> <option value="Name">שם</option> <option value="Grade">ציון</option> </select> <button type="submit">מיין</button> </form>

2. סינון — WHERE LIKE

public void OnPostFilter() { string query = "SELECT * FROM Students WHERE Name LIKE @search"; SqlCommand cmd = new SqlCommand(query, con); cmd.Parameters.AddWithValue("@search", "%" + SearchText + "%"); // Fill DataTable... }
אבטחה!

תמיד השתמשו ב-SqlParameter ולא בשרשור מחרוזות. "%" + SearchText + "%" בטוח כי הוא עובר כפרמטר.

3. מיון — ORDER BY

public void OnPostSort() { // Whitelist — מגן מפני SQL Injection! var allowed = new[] { "Name", "Grade", "City" }; string col = allowed.Contains(SortBy) ? SortBy : "Name"; string query = $"SELECT * FROM Students ORDER BY {col}"; // Fill DataTable... }
למה Whitelist?

שם עמודה לא ניתן להעביר כ-SqlParameter. לכן בודקים שהערך נמצא ברשימה מותרת — אחרת משתמש זדוני יכול להזריק SQL.

4. שילוב סינון + מיון
